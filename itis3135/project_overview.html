Project Overview & Design Document

Project Name

CodeCraft: Front-End Basics Tutor

Project Overview

  • Purpose: To design and develop an interactive educational web application for a peer student who wants to learn the fundamental concepts of HTML, CSS, and JavaScript.
  • Intended Users: Beginners and peer classmates with little to no coding experience who want a guided, hands-on introduction to web development.
  • Content Overview: The site will feature structured learning modules, visual examples of CSS styling, a glossary of coding terms, a dynamic quiz to test their knowledge, and a contact page for tutoring assistance.

Page Design Details

1. Home Page / Dashboard (index.html)

  • Purpose: To welcome the student and provide an overview of the curriculum.
  • Audience: The student user.
  • Content: A friendly introduction, course objectives, and quick links to the HTML, CSS, and JS modules.
  • Data Entry: None.
  • Actions: Navigation clicks to begin the lessons.

2. CSS Visualizer Lesson (css_lesson.html)

  • Purpose: To teach the basics of Cascading Style Sheets through visual examples.
  • Audience: The student user.
  • Content: Explanations of properties like color, padding, margins, and flexbox.
  • Data Entry: None.
  • Actions: Users will interact with buttons that trigger JavaScript to change the CSS of elements on the page in real-time (e.g., clicking a button to change a box from blue to red).

3. JavaScript Basics (js_lesson.html)

  • Purpose: To introduce core programming logic like variables, functions, and events.
  • Audience: The student user.
  • Content: Code snippets, explanations of syntax, and a small interactive calculator or counter example.
  • Data Entry: Yes (Basic text inputs for the interactive examples).
  • Actions: Users can click buttons to run simple predefined JavaScript functions (like triggering an alert or adding two numbers) to see how JS interacts with HTML.

4. Knowledge Check Quiz (quiz.html)

  • Purpose: To assess what the student has learned from the modules.
  • Audience: The student user.
  • Content: A multiple-choice quiz covering HTML tags, CSS properties, and JS concepts.
  • Data Entry: Yes (Radio buttons and checkboxes for quiz answers).
  • Validations: JavaScript will ensure all questions are answered before allowing submission.
  • Actions: Clicking "Submit" will instantly calculate their score and display a dynamic results message with feedback.

5. Ask the Tutor (contact.html)

  • Purpose: To allow the student to request help or ask specific coding questions.
  • Audience: The student user.
  • Content: A contact form and resources/links for further learning (like MDN or freeCodeCamp).
  • Data Entry: Yes (Name, Email, Module they are stuck on, Message).
  • Validations: Standard HTML5 required fields and email formatting.
  • Actions: Submission button processes the form data and displays a simulated "Message Sent" confirmation.

Dynamic Functionality

  • Real-Time CSS Toggles (css_lesson.html): I will use JavaScript to allow the user to toggle CSS classes on a set of sample HTML elements. Purpose: To visually demonstrate how CSS properties (like `border-radius` or `background-color`) alter the DOM instantly. Example: CSS-Tricks interactive snippets.
  • Instant Quiz Grader (quiz.html): I will use JavaScript to read the values of the selected radio buttons on the form, calculate a score, and dynamically inject the results (e.g., "3/5 Correct!") onto the page without refreshing. Purpose: To provide immediate feedback to the student to reinforce learning.
  • Accordion FAQ/Glossary (index.html or js_lesson.html): I will use a jQuery UI Accordion widget to display a list of common coding terms. Purpose: To keep the page clean and organized by hiding definitions until the user clicks on a specific term to reveal the text.
